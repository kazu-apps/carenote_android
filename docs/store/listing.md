**詳細説明** (4000文字以内):
```
ケアのては、家族の介護をもっと安心に、もっとスムーズにするための記録アプリです。

毎日の服薬管理、健康状態の記録、通院やデイサービスの予定管理、やるべきタスクの整理、そして家族間の申し送りメモまで、介護に必要な情報をひとつのアプリにまとめました。

■ 主な機能

【服薬管理】
・薬の名前、用量、服用タイミング（朝・昼・夕）を登録
・毎日の服薬記録（飲んだ・飲めなかった・後で通知）
・服薬履歴の確認

【健康記録】
・体温、血圧、脈拍、体重、食事量、排泄、体調メモを記録
・グラフ表示で体調の変化を一目で把握（7日間・30日間）
・異常値の基準を設定して見守り

【カレンダー】
・通院、訪問介護、デイサービスなどの予定を管理
・月表示カレンダーで予定を一覧
・終日イベント、時間指定イベントに対応

【タスク管理】
・買い物、掃除、書類手続きなど日常の介護タスクを管理
・優先度（高・中・低）と期限の設定
・完了/未完了のフィルタ表示

【メモ・申し送り】
・介護に関する気づきや連絡事項を記録
・タグ（体調・食事・報告・その他）で分類
・検索機能でメモをすぐに見つけられる

■ 特長

・オフライン対応 — インターネット不要、すべてのデータは端末内に保存
・プライバシー重視 — 外部サーバーへのデータ送信なし、広告なし、トラッキングなし
・高齢者にやさしいデザイン — 大きな文字、広いタッチ領域、見やすい配色
・日本語・英語対応

■ こんな方におすすめ

・離れて暮らす親の介護をしている方
・家族で介護を分担している方
・デイサービスや訪問介護の予定を管理したい方
・毎日の服薬や健康状態を記録したい方

■ 注意事項

本アプリは医療機器ではなく、医療上の助言を提供するものではありません。
健康に関する判断は、必ず医療専門家にご相談ください。
```

## 4. コンテンツレーティング / Content Rating (IARC)

Google Play Console の「コンテンツのレーティング」質問票への回答ガイド。

| 質問 | 回答 | 理由 |
|------|------|------|
| 暴力的なコンテンツ | なし | 介護記録アプリ |
| 性的コンテンツ | なし | 介護記録アプリ |
| 言語 | なし | 不適切な言語表現なし |
| 規制物質 | 医薬品への言及あり | 服薬管理機能で薬の名前・用量を記録 |
| ギャンブル | なし | ギャンブル要素なし |
| ユーザー間のやり取り | なし (v1.0) | オフラインアプリ、ユーザー間通信なし |
| 個人情報の収集 | 端末内のみ | 外部サーバーへの送信なし |
| 広告 | なし | 広告表示なし |
| 暗号通貨 | なし | 暗号通貨機能なし |

**想定レーティング**: Everyone (全ユーザー対象)

> **注意**: 「規制物質」の項目で「医薬品への言及」を選択する必要がある。
> これは服薬管理機能で薬の名前を入力・表示するため。
> ただし、薬の推奨・販売は行わないため、レーティングへの影響は軽微。

## 11. チェックリスト / Pre-Submission Checklist

Google Play Console への入力時に確認するチェックリスト。

### アプリの設定
- [ ] アプリ名を入力 (日本語 + 英語)
- [ ] 短い説明を入力 (日本語 + 英語)
- [ ] 詳細説明を入力 (日本語 + 英語)

### グラフィック
- [ ] アプリアイコン (512x512) をアップロード
- [ ] スクリーンショット (5〜8枚) をアップロード
- [ ] フィーチャーグラフィック (1024x500) をアップロード

### カテゴリとタグ
- [ ] アプリカテゴリを「医療」に設定
- [ ] タグを設定

### コンテンツのレーティング
- [ ] IARC 質問票に回答
- [ ] レーティング結果を確認

### プライバシーとポリシー
- [ ] プライバシーポリシー URL を設定
- [ ] データセーフティセクションを入力
  - [ ] データ収集: なし (ネットワーク送信なし)
  - [ ] データ共有: なし
  - [ ] セキュリティ: データは端末内に暗号化保存 (Room + SQLCipher なし、OS レベル暗号化)
  - [ ] データ削除: アプリのアンインストールで全データ削除

### 対象ユーザー
- [ ] ターゲット年齢層: 18歳以上
- [ ] 子ども向けではないことを確認

### リリース管理
- [ ] 署名済み AAB/APK をアップロード
- [ ] リリースノートを入力 (日本語 + 英語)
- [ ] 内部テスト → クローズドテスト → オープンテスト → 製品版 の順で公開

### 連絡先
- [ ] デベロッパーメールアドレスを設定
- [ ] (任意) ウェブサイト URL を設定
